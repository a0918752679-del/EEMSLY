# AI 輔助功能說明

## 已整合功能

1. AI 初核資料品質檢查
   - 車號格式
   - 案件編號空白提醒
   - 背景修正後分貝與管制標準比對
   - 50km/h 以下對應 86dB、50–70km/h 對應 90dB 的提醒
   - 量測日期、地點缺漏檢查
   - 同車號重複案件提醒

2. AI 案件優先排序
   - 依超標幅度、案件狀態、重複車號、資料錯誤、行政待辦、通檢期限計分
   - 分為 S / A / B / C 級

3. AI 行政待辦
   - 已核可但未填公文文號
   - 告發案件未填裁處字號
   - 通檢期限 7 日內提醒
   - 已發文但未填送達日期

4. AI 案件摘要
   - 每筆案件可點「AI摘要」
   - 顯示案件摘要、品質檢查、優先等級、處理建議
   - 可一鍵複製 AI 建議退回原因

5. LINE 查詢模擬與 Webhook
   - 前端可用「LINE 查詢模擬」測試
   - 後端提供 `POST /line/webhook`
   - 支援：查車號、查案件編號、今天待審案件、行政待辦

## AI_PROVIDER 說明

### heuristic

預設模式，不需要 AI API Key。
適合正式上線第一階段，穩定、可控、不會把資料送外部模型。

### openai

設定：

```env
AI_PROVIDER=openai
AI_MODEL=gpt-4.1-mini
OPENAI_API_KEY=你的OpenAI_APIKey
```

啟用後，系統仍先用規則做初核，再把脫敏後的案件資料送到 OpenAI 生成較自然的案件摘要、退回原因與 LINE 回覆。

## 個資控管

OpenAI 模式下，系統只送脫敏資料：

- 車號會遮蔽為前 2 碼 + **** + 後 2 碼
- 不送車主姓名
- 不送證號
- 不送生日
- 不送完整戶籍地
- 僅送案件類型、狀態、日期、地點摘要、分貝、標準、行政階段

AI 不會自動核可、退回、裁罰或結案；所有決策仍由承辦與長官確認。
